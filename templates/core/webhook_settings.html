    How Webhooks Work
    Important: Portainer provides webhook URLs that external systems can call to trigger updates. Your Django app can call these URLs to update your stack.
    Using Portainer's Webhook URL
    1
    In Portainer, go to your stack details
    2
    Copy the webhook URL (like: https://portainer.errorlo[...]1b-4be5-ac30-2a172d332b80)
    3
    Use this URL in your Django app to trigger stack updates
    Example: Call Portainer's webhook URL to update your stack
    URL: [Portainer's webhook URL]
    Method: POST
    No authentication required - Portainer webhook URLs are public endpoints
    Environment Variables

    Add these environment variables to your Django settings:

    PORTAINER_URL=http://your-portainer:9000
    PORTAINER_USER=your-username
    PORTAINER_PASSWORD=your-password
    PORTAINER_STACK_NAME=your-stack-name
    PORTAINER_WEBHOOK_SECRET=your-webhook-secret
    Testing the Webhook

    You can test the webhook manually using curl:

    curl -X POST {{ webhook_url }} \
      -H "Content-Type: application/json" \
      -H "X-Portainer-Signature: [your-signature]" \
      -d '{"event": "stack_update", "stack": "{{ stack_name }}"}'
